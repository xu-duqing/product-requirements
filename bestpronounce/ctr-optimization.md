# CTR 优化需求文档

**版本**: v1.0
**日期**: 2026-03-03
**优先级**: P0

---

## 问题背景

当前搜索曝光高但点击率极低，核心问题是：**搜索结果页没有提供"发音"信息，用户无法判断是否满足需求**。

### 数据现状

| 搜索词 | 曝光 | 点击 | CTR |
|--------|------|------|-----|
| atorvastatin | 322 | 0 | 0% |
| cómo se pronuncia información | 613 | 1 | 0.16% |
| wegovy aussprache | 68 | 2 | 2.94% |
| cerave怎么读 | 20 | 1 | 5% |

用户搜索"发音"，但在点击前得不到任何"怎么读"的答案。

---

## 核心目标

**在搜索结果页直接提供发音信息，降低用户决策成本。**

---

## 产品策略

### 理念
- 做少但极致
- 用户不需要点进来就能听到发音
- 每一个交互都要有明确价值

---

## 功能需求

### P0 - 热门词卡片增强

**当前状态**: 纯文本按钮

**目标状态**: 信息卡片，包含：
- 词名
- 音标 / 拼音
- 语言标签 (EN/ZH/JP/ES/DE)
- 微型播放按钮

**交互**:
- 点击 → 跳转详情页
- 点击播放按钮 → 播放完整发音

---

### P1 - 搜索结果卡片

**场景**: 当搜索返回多条结果时（当前未实现，预留）

**需求**:
- 每条结果显示为卡片
- 包含音标和语言标签
- 鼠标悬停 → 自动播放 1 秒预览

---

### P2 - 实时搜索建议

**场景**: 用户输入时

**需求**:
- 下拉建议显示：
  - 匹配词
  - 音标
  - 语言标签
- 键盘上下选择，回车确认

---

## 技术需求

### API 改动

**1. word-info 接口扩展**
```json
{
  "word": "atorvastatin",
  "phonetic": "a-TOR-va-sta-tin",
  "language": "en",
  "preview_url": "/api/tts-preview?word=atorvastatin&lang=en"
}
```

**2. 新增热门词接口**
```http
GET /api/popular-words?lang=en

Response:
[
  {
    "word": "atorvastatin",
    "phonetic": "a-TOR-va-sta-tin",
    "language": "en",
    "search_count": 322
  }
]
```

---

### 前端改动

**1. HomePage.vue**
- 热门词区域从 `button` 改为 `WordCard` 组件
- 新增 `WordCard.vue` 组件

**2. WordCard.vue**
```
┌─────────────────────────┐
│ atorvastatin            │
│ /a-TOR-va-sta-tin/  🔊  │
│ [English]               │
└─────────────────────────┘
```

**3. 语言检测**
- 搜索 `wegovy aussprache` → 识别为德语
- 搜索 `cerave怎么读` → 识别为中文
- 自动切换 UI 语言标签

## 不做的功能

- ❌ 用户评价系统
- ❌ 收藏夹

## 成功指标

### 短期（2周）
- 热门词 CTR 从 ~2% 提升到 15%
- 热门词播放率 > 30%

### 中期（1月）
- 整体 CTR > 10%
- 用户停留时间 +20%

## 风险与缓解

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|----------|
| TTS 接口 QPS 不足 | 中 | 高 | 缓存热门词音频，限制预览时长 |
| 音标数据不准确 | 低 | 中 | 优先用 API 数据，降级到自动生成 |
| 多语言识别错误 | 中 | 中 | 关键词词典匹配 + fallback 到英语 |
