# 成长资本（Growth Capital）v3.1 - 极简版产品需求文档

**版本**: v3.1
**日期**: 2026-03-10
**优先级**: P0
**平台**: 移动端 H5

---

## 核心理念

**极简主义 - 做少，但做到极致**

- 不模拟企业管理
- 不做复杂角色系统
- 不强行区分家长端 / 孩子端

这是一个**家庭共享的项目账本**：
- 一个账号 = 一个家庭账本
- 家长和孩子一起看，一起操作
- 系统不关心“谁是家长、谁是孩子”，只关心“项目”和“记录”

**核心关系：**

> 想做什么项目 → 项目目标 → 需要多少资金 → 阶段性完成 → 项目完成 → 记一笔成长资本

---

## 版本对比

### v2.0（复杂版）
- 流程：5步（发起→审核→执行→验收→发放）
- 页面：25 个
- 分离：家长端 / 孩子端
- 功能：项目管理、账户系统、价值分配、成就系统...

### v3.0（极简版初稿）
- 流程：3步（设置目标→完成→确认）
- 页面：8 个（家长 5 个 + 孩子 3 个）
- 分离：家长端 / 孩子端
- 功能：目标、确认、记录、统计

### v3.1（当前版本）⭐
- 流程：3步（想做什么 → 项目目标 → 需要多少资金）
- 页面：4 个（统一界面）
- 不再区分家长端 / 孩子端
- 项目成为一等公民（有详情页、阶段、状态）
- 功能：项目列表、项目详情、记录、统计

---

## 产品定位

**产品名称**: 成长资本（Growth Capital）- 极简版

**一句话：**
> 帮家庭把“孩子做过哪些项目、创造过哪些价值”记清楚。

**目标用户**:
- 使用者：家庭（家长 + 孩子）
- 使用场景：
  - 孩子有想做的项目
  - 家长愿意以“项目”的方式来奖励/记账
  - 不需要复杂的财务系统

---

## 核心流程

### 新流程（v3.1）

```text
Step 1：你想完成一个什么样的项目？
Step 2：这个项目的目标是什么？
Step 3：你需要多少资金帮助你完成这个项目？
→ 项目开始 → 阶段性完成 → 项目完成 → 生成一条收入记录
```

---

## 功能模块

- 项目系统（创建、列表、详情、阶段性完成、整体完成）
- 记录系统（收入记录、支出记录）
- 统计系统（项目数量 & 累计价值）
- 账号系统（家庭账本，极简登录）

## 一、账号系统（家庭账本）

### 1.1 注册 & 登录

**设计原则：**

- 一个账号 = 一个家庭账本
- 不区分“家长账号 / 孩子账号”
- 使用手机号 + 昵称注册

**字段：**
- 手机号（必填）
- 昵称（必填）

**接口示例：**
```http
POST /api/auth/register
POST /api/auth/login
GET  /api/auth/me
```

> 后续如果要支持“多个孩子”，可以在同一个账本下增加“成员”概念，但 v3.1 不设计多成员结构，只做单账本。

## 二、项目系统（核心）

### 2.1 项目创建流程（三步）

入口：`+ 新建项目`

#### Step 1：你想完成一个什么样的项目？

**意图：** 先说“想做什么”，而不是先谈钱。

**字段：**
- `项目名称（必填）`
  - 示例：
    -「做一次家庭旅游规划」
    -「数学从 80 分提高到 90 分」
    -「做一只会走路的机器人」

- `项目类型（可选，下拉）`
  - 学习 / 创造 / 服务 / 生活 / 其他

- `项目想法（必填，简短描述）`
  - 示例文案提示：
    -「我想通过这次项目学会 ……」
    -「我想帮家里解决 ……」

**交互：**
- 底部按钮：`下一步`

---

#### Step 2：这个项目的目标是什么？

**意图：** 把“模糊的想做”变成“清晰的目标”。

**字段：**
- `项目目标（必填）`
  - 示例：
    -「端午三天两晚自驾行程，行程+预算+预订全部搞定」
    -「期末数学成绩从 80 分到 90 分」
    -「做出一个可以走 3 米不摔倒的机器人」

- `完成标准 / 验收方式（可填）`
  - 示例提示：
    -「一份写好的行程单 + 机酒订单截图」
    -「期末试卷 + 平时练习记录」
    -「录一段机器人走路的视频」

- `预估用时（可选）`
  - 输入：几天 / 几周（简单整数即可）

**交互：**
- 底部按钮：`上一步` / `下一步`

---

#### Step 3：你需要多少资金帮助你完成这个项目？

**意图：** 让孩子开始练习“预算”和“价值感”。

**字段：**
- `申请预算总额（必填，数字）`
  - 示例：`150 元`

- `预算用途说明（可选，文本）`
  - 示例：
    -「资料 50 / 工具 50 / 其他 50」
    -「打印资料 20 / 小工具 30 / 车费 100」

- `预计奖励价值（必填，数字）`
  - 定义：项目按标准完成后，记入账本的“成长资本”金额
  - 示例：`50 元`

**交互：**
- 底部按钮：`上一步` / `创建项目`

---

### 2.2 项目数据结构（概念）

```ts
interface Project {
  id: string;

  // Step 1
  title: string;       // 项目名称
  category?: 'learning' | 'creation' | 'service' | 'life' | 'other';
  idea: string;        // 项目想法

  // Step 2
  goalDescription: string; // 项目目标
  checkStandard?: string;  // 完成标准 / 验收方式
  expectedDuration?: string; // 预估用时（如："2周"）

  // Step 3
  requestedBudget: number;    // 申请预算
  budgetBreakdown?: string;   // 预算用途说明
  expectedRewardValue: number; // 完成后的成长资本

  // 状态
  status: 'active' | 'in_progress' | 'partially_done' | 'completed' | 'archived';
  createdAt: Date;
  updatedAt: Date;
}
```

---

### 2.3 项目列表

**需求：**
- 列出所有项目（卡片形式）
- 支持筛选：全部 / 进行中 / 已完成 / 已归档
- 卡片展示关键信息：
  - 项目名称
  - 当前状态
  - 预计奖励价值

**示意：**

```text
[项目列表]

进行中 (2)
- 做一次家庭旅游规划        状态：进行中   预计奖励：50 元
- 数学从 80 分到 90 分      状态：阶段完成 预计奖励：30 元

已完成 (1)
- 做一只会走路的机器人      状态：已完成   奖励：80 元
```

点击某一项目，进入“项目详情页”。

---

### 2.4 项目详情页（支持阶段性完成）

> 入口：项目列表点击某一条

**信息结构：**

1. **项目概览**
   - 标题 + 类型
   - 当前状态：进行中 / 阶段完成 / 已完成 / 已归档
   - 关键数字：申请预算、预计奖励

2. **项目内容**
   - 项目想法
   - 项目目标
   - 完成标准 / 验收方式
   - 预估用时

3. **项目阶段（最多 3 个）**

极简预设阶段模板：
- 准备阶段：查资料 / 看攻略 / 做计划
- 执行阶段：真正去做这件事
- 收尾阶段：整理成果 / 写总结 / 做分享

```ts
interface ProjectStage {
  id: string;
  title: string;           // 阶段名称
  description?: string;    // 阶段说明
  status: 'pending' | 'done';
  completedAt?: Date;
}

interface Project {
  ...
  stages?: ProjectStage[];
}
```

**详情页中展示方式：**

```text
[阶段进度]

✓ 准备阶段
  - 查好 3 个备选目的地
  完成时间：2026-03-18

□ 执行阶段
  - 决定最终目的地，做出每天行程和预算

□ 收尾阶段
  - 做一份行程文档，给家里讲一遍
```

**操作：**
- “标记阶段完成”：勾选某阶段 → `status = done`
- 如果至少一个阶段完成，则项目整体可显示为 `partially_done`
- “标记项目完成”：当所有阶段完成后，可以点“项目完成”按钮

4. **项目完成 & 记账**

- 当点击“项目完成”：
  - 项目状态：`status = completed`
  - 自动生成一条收入记录：

```ts
interface Record {
  id: string;
  type: 'income';                 // 项目完成只记收入
  amount: number;                 // = expectedRewardValue
  description: string;            // = 项目名称
  projectId: string;              // 关联项目
  createdAt: Date;
}
```

> 项目完成后，仍然可以在“项目详情”里查看完整过程，只是状态从“进行中”变为“已完成”。

## 三、记录系统

### 3.1 收入记录

来源：
- 项目完成时，系统自动生成收入记录

展示：
- 时间、金额、来源项目

```text
[记录列表]

收入
- 03-10  +50 元  做一次家庭旅游规划
- 03-08  +30 元  数学从 80 分到 90 分
```

### 3.2 支出记录

来源：
- 家长手动添加支出

字段：
- 金额
- 说明

---

## 四、统计系统

**轻量统计，不做复杂图表：**
- 项目维度：
  - 进行中项目数量
  - 已完成项目数量
- 价值维度：
  - 累计收入（所有项目完成产生的成长资本）
  - 累计支出
  - 当前余额（收入 - 支出）

---

## 五、页面结构（统一界面）

1. 项目列表页
2. 项目详情页
3. 记录列表页
4. 统计页

所有页面家长和孩子看到的内容一致，区别只在于实际操作习惯：
- 由谁发起项目
- 由谁点击“阶段完成”
- 由谁点击“项目完成”

系统不过多干预，由家庭自行约定。

---

## 六、成功标准（v3.1）

- 一个家庭可以用它完整记录：
  - 至少 3 个项目，从想法 → 目标 → 预算 → 完成 → 记账
- 孩子可以自己说清楚：
  - “我现在在做哪个项目？”
  - “我完成了哪一步？”
  - “这个项目完成之后，我会获得多少成长资本？”
- 家长觉得：
  - 操作不啰嗦
  - 比记在 Excel 里更舒服一点
  - 不会被规则和流程“框死”
